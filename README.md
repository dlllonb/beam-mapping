# beam-mapping

Python analysis suite for GRASP-simulated beam maps from a 331-element CMB horn array at 150 GHz.

This repo reproduces key beam-map diagnostics originally written in IDL — polarization-angle rotation, Mueller matrix leakage, differential beam mismatch, and rough B-mode contamination estimates — rewritten in Python with matplotlib output.

---

## Folder structure

```
beam-mapping/
├── scripts/                   User-facing runnable scripts
│   ├── run_array_map.py       Batch-process all feeds → array_data.npz
│   ├── make_array_plots.py    Generate 7-page summary PDF from array_data.npz
│   ├── make_diagnostic_plots.py  Key science diagnostic figures (rotation, leakage, mismatch)
│   ├── run_single_beam.py     Analyse one feed (Mueller maps, beam cuts, CMB sims)
│   ├── make_bmode_plot.py     B-mode contamination estimate vs CMB theory
│   └── make_mode_fraction_plot.py  Aperture efficiency sweep
│
├── core/                      Internal library — not meant to be run directly
│   ├── config.py              Instrument constants and data paths
│   ├── read_grd.py            TICRA/GRASP .grd file parser
│   ├── read_horn_layout.py    Horn position loader + sky-offset computation
│   ├── mueller.py             Jones-to-Mueller matrix construction
│   ├── beam_utils.py          Beam characterisation (FWHM, ellipticity, etc.)
│   └── cmb_sim.py             CMB power spectra + Gaussian sky realisations
│
├── data/
│   ├── horn_layout/           Horn position table (tracked)
│   │   └── horn_layout_331.dat
│   ├── cosmological_models/   CAMB power spectra for r in {0.001, 0.01, 0.05, 0.1, 0.2} (tracked)
│   └── maps/                  Beam map .grd files — NOT tracked (see Data section)
│
├── outputs/                   Generated PDFs, PNGs, .npz files — NOT tracked
├── requirements.txt
└── .gitignore
```

## Data

### Beam maps (`data/maps/`)

The beam map files are large (~3 GB total) and are not tracked by git. Place the GRASP `.grd` files under:

```
data/maps/150_GHz_v26_maps/
```

Expected naming convention: `beam_map_331F{N}_pol1.grd` and `beam_map_331F{N}_pol2.grd` for each feed N (331 feeds, 2 polarisations each = 662 files).

### Other data files

The horn layout (`data/horn_layout/horn_layout_331.dat`) and CAMB cosmological models (`data/cosmological_models/`) are small and **are** tracked by git — no extra steps needed.

## Quick start

### Step 1 — Process all feeds (batch, slow)

Reads every pol1/pol2 `.grd` pair, computes Mueller matrices and beam metrics, writes a single compressed `.npz`:

```bash
python scripts/run_array_map.py \
  --maps_dir data/maps/150_GHz_v26_maps/ \
  --out outputs/array_data.npz
```

This produces `outputs/array_data.npz` (~1.2 GB), the input for all subsequent plotting scripts.

### Step 2 — Key science diagnostic figures

Reproduces the main polarization and Mueller diagnostics:

- Focal-plane polarization-angle rotation map
- Rotation angle histogram (all 331 feeds)
- Integrated Mueller leakage maps: QT, UT, QU, UQ
- Differential beam mismatch: centre offset, FWHM, ellipticity, orientation
- Representative per-feed Mueller beam maps

```bash
python scripts/make_diagnostic_plots.py outputs/array_data.npz \
  --output-pdf outputs/diagnostic_figures.pdf
```

Or save individual PNGs:

```bash
python scripts/make_diagnostic_plots.py outputs/array_data.npz \
  --output-dir outputs/diagnostic_figures/
```

### Step 3 — Summary array plots

Generates a broader 7-page PDF covering Mueller scatter plots, averaged beam maps, FWHM/ellipticity/gain across the array, and histograms:

```bash
python scripts/make_array_plots.py \
  --data outputs/array_data.npz \
  --out outputs/array_plots.pdf
```

### Single-feed analysis

Analyse one feed individually (Mueller maps, beam cuts, optional CMB convolutions):

```bash
python scripts/run_single_beam.py \
  --pol1 data/maps/150_GHz_v26_maps/beam_map_331F1_pol1.grd \
  --pol2 data/maps/150_GHz_v26_maps/beam_map_331F1_pol2.grd \
  --out outputs/feed_F1/
```

Add `--realizations 10 --bmodes` to run CMB sky simulations (needed for `make_bmode_plot.py`).

### B-mode contamination estimate

Compare systematic B-mode power from two feeds against CMB theoretical curves. Requires `run_single_beam.py` output directories run with `--realizations`:

```bash
python scripts/make_bmode_plot.py \
  --feed1 outputs/feed_F1/ \
  --feed2 outputs/feed_F100/ \
  --out outputs/bmode.pdf
```

## Notes

- This is a Python translation of the original IDL beam-mapping workflow developed by Brad Johnson. The math has been preserved from the working `plotting_py/` version.
- All scripts resolve paths relative to their own location (via `Path(__file__)`), so they can be run from any working directory.
- Generated outputs and large beam map data are intentionally not tracked by git. The `.gitignore` excludes `outputs/`, `data/maps/*`, and all `.grd`, `.npz`, `.pdf`, `.png` files.
- The `data/maps/.gitkeep` and `outputs/.gitkeep` files preserve the directory structure in git without tracking any data.
